Evaluation Metadata

The next step is to report the conformance of the publication and provide information about the testing organization/individual in the Conformance Metadata tab.

Note: Do not skip this step just because a publication has failed its accessibility evaluation. It is important to note who carried out the evaluation and, if possible, to provide a link to the detailed report explaining why.

First, indicate whether the publication has passed or failed the conformance requirements. If it passes, select whether it passed WCAG Level A or AA.

Next, provide the name of your organization.

If available, you can also specify the URL where the final report will be located. This metadata is not included in the final report, but if you use the option at the bottom of the page to generate metadata for the EPUB publication, providing the link will allow the report to be discovered from the publication.

Finally, it is possible to include the name and link to a certification credential.

Note: It is not necessary to include DAISY certification in this field. A credential is included in the generated report.

To generate a set of metadata that can be pasted into the EPUB publication, use the Generate button at the bottom of the page. Metadata compatible with the EPUB version will be output to the text box at the bottom of the page.

If any validation errors are encountered, an alert dialog will appear. Errors will be listed in an error panel that opens at the bottom of the page. Click on a message to go to the location of the error.
