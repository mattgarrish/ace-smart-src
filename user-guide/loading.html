Loading the Ace JSON Report

To prepare this site to evaluate a publication, it is first necessary to load the JSON report output by the Ace tool. This report is in the file called report.json. It is not the HTML file that you use to review the issues found by that tool.

To load the JSON file, you can drag it from your file system and drop it in the blue box in the Start tab:

If you are not able to drag and drop the report, it is also possible to select it from the file system.

Loading the report through the drag-and-drop field sends the selected file to the SMART server, however no information about your publication is stored (the data is simply read and returned to the application, as JavaScripts cannot read local files).

If you prefer not to send the Ace report over the internet, a local file loading option is provided.

Regardless of which file loading method you choose, the SMART tool will attempt to customize itself based on the information in the provided report. When it completes, an alert will be generated indicating what optimizations have been made, if any.

Note: If automatic detection results in incorrect tests being disabled, they can be re-enabled in the Conformance Verification tab.

Note: If accessibility metadata is already present in the publication, the SMART tool will not attempt to infer additional metadata.

If you need to review the load report again after closing the dialog, click on the link below the load box.
