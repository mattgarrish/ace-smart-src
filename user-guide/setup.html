EPUB Accessibility Conformance and Reporting Tool

Evaluation Setup

Loading the Ace JSON Report

To prepare this site to evaluate a publication, it is first necessary to load the JSON report output by the Ace automated tool. This report is in the file called ace.json. It is not the HTML file that you use to review the issues found by that tool.

To load the JSON file, you can drag it from your file system and drop it in the blue box in the Start tab:

If you're not able to drag and drop the report, it is also possible to select it from the file system.

The reporting tool will attempt to load the JSON report and customize itself based on the information it provides. When it completes, an alert will be generated indicating what optimizations have been made, if any.

If automatic detection results in incorrect tests being disabled, they can always be re-enabled by going to the configuration tab.

Note: The process for loading an Ace report is the same as for loading a saved report.

EPUB Format Configuration

A field to select the format of the EPUB publication is located below the report loading box.

The format selected does not change the evaluation criteria, but it does control optional metadata that can be generated from the tool (i.e., discovery and conformance metadata for the package document). It is also the reported format of the publication in the final report.

WCAG Configuration

The reporting tool is configured by default to verify that publications meets WCAG Level A requirements.

To check for Level AA conformance, simply switch the setting. The list of success criteria to verify in the next step automatically updates to match the selected setting.

The first section allows you to WCAG Level AA and AAA success criteria to the conformance verification tab.

Showing these criteria does not change the conformance level being verified, and their status is not checked when the final report is generated. Showing them allows you to indicate that a publication meets requirements beyond the base level being tested. If any of these criteria are set to pass or fail, they will be included in the final report.

Note: To change the WCAG conformance level the publication has to meet, use the options on the Start tab.

It is also possible to hide content-specific success criteria if you know that they do not apply to the publication (e.g., scripting requirements).

These options are automatically updated to reflect the content when the Ace JSON report is loaded in the setup step.

Note that deselecting an option only hides the related criteria from view. The criteria are still included in the generated report with a "not applicable" status.

Note: The reporting tool does not include the option to conform to level AAA, as no content can realistically conform to all the requirements of this level.

Publication Information

The final section on the Start tab is dedicated to metadata about the publication. This section can be skipped if you do not intend to generate a final report, otherwise you need to provide at least the title of the publication.

The title, author(s), identifier, publisher and last modified dates are all used to accurately identify the publication being reported on.

In addition to the provided fields, it is also possible to add optional metadata using the text box at the end of the form. To add optional metadata, type a human-readable label followed by a colon and space and then add the value.

Note: As the optional metadata is for human consumption, it is not recommended to use property names (e.g., use a label like "Created On" instead of "dcterms:created").
