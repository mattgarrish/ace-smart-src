Overview

There are two steps in the accessibility evaluation process for EPUB publications:

1. Automated Verification

The first step is to ensure the publication passes automated testing:

  1. Use the epubcheck validator to check the publication for errors to the EPUB specification. Although this tool does not specifically perform accessibility checks, it ensures the publication is a valid EPUB publication, which is necessary to pass WCAG success criterion 4.1.1.
  2. Use the Ace accessibility tool to check accessibility requirements that can be programmatically determined. The JSON file output by this program is used to populate the SMART interface.
2. Manual Verification

Although automated validation tools will alert you to problems that can be programmatically determined, ensuring accessibility conformance also requires a number of manual checks. There are many aspects of accessibility that a tool cannot verify (quality of descriptions, scripting accessibility, keyboard issues, etc.). These checks must be performed to ensure conformance with the EPUB Accessibility specification — a claim of conformance cannot be made only from automated testing.

The Ace SMART tool is designed to help you perform these manual assessments, sometimes using the Ace HTML report to simplify verification.

Note that in order to carry out an evaluation, editing software that allows you to review the resources of the publication is required. This is true even if you are only reporting issues and not fixing them, as you need access to the raw markup of the publication to verify many accessibility requirements.

The EPUB publication does not have to be zipped to carry out the manual evaluation steps, as zipping does not directly affect the accessibility.
