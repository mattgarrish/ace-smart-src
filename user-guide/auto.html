Automated Evaluation

Before beginning the manual evaluation process, ensure that the publication passes the Ace automated evaluation tool without error. (If errors cannot be fixed, they will be reported under the appropriate WCAG success criterion in the SMART interface.)

Instruction on how to run Ace and interpret its reports is available from the project home page. The latest version of the tool can be downloaded from the releases page.

When the Ace tool is run, it generates two primary documents:

report.html
The HTML report document provides a human-readable report of any issues found, as well as supplementary information like the images, audio and video. To expedite manual evaluations, it will be necessary to refer to the additional information provided in this report, so keep the report open as you proceed.
report.json
The JSON report document provides machine-readable data about the automatic evaluation. It is loaded into the SMART interface to prepare and simplify the manual evaluation process.

Note: Although it is possible to evaluate a publication without using the Ace automated evaluation tool, it is not recommended. The automated tool cannot check every failure, but it prevents oversights of the common errors it can detect.
