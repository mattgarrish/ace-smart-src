Automated Evaluation

Before beginning the manual evaluation process, ensure that your publication passes the Ace automated evaluation tool without error.

Note: If you are only creating an evaluation report, any unfixed Ace errors will be listed under the appropriate WCAG success criterion.

Instruction on how to run Ace and interpret its reports is available from the project home page. The latest version of the tool can be downloaded from the releases page.

When the Ace tool is run, it generates two primary documents:

report.html
The HTML file provides a human-readable report of any issues found, as well as supplementary information like lists of images, audio and video and visualizations of the data structures of the publication. This additional information helps expedite certain manual inspection processes (e.g., checking whether images are decorative or have meaningful alternative texts), so keep the report open as you proceed with a SMART evaluation.
report.json
The JSON document provides machine-readable data about the automatic evaluation. This is the file that is loaded into the SMART interface to prepare and simplify the manual evaluation process.

Note: Although it is possible to evaluate a publication without using the Ace automated evaluation tool, it is not recommended. The automated tool cannot check every failure, but it prevents oversights of the common errors it can detect.
