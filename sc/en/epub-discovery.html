Open the package document and verify that all of the following required discovery metadata properties are set and the value(s) they specify are correct for the publication:

schema:accessMode
  • Check that each access mode matches the way in which information is encoded.
  • Check that access modes are not set when the content is presentational.
  • Chcek that no access modes have been missed.
schema:accessibilityFeature
  • Check that features are not incorrectly listed as present.
  • Check that features have not been missed.
schema:accessibilityHazard
  • Check that hazards are not incorrectly listed as present.
  • Check that hazards have not been missed.
schema:accessibilitySummary

Check that the summary provides an accurate description of the accessibility of the publication (e.g., it identifies conformance and notes any special information the general metadata cannot capture).

Note: It is recommended, but not required, that the schema:accessModeSufficient property be specified.

Caution: Metadata listed in the Discovery tab may not reflect what is actually in the package document (e.g., the SMART tool inferred metadata during the import of an Ace report, or you have manually updated the metadata and not saved it back to the publication). Always check the package document when verifying this success criterion.
